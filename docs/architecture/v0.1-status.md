## Data storage

- **SQLite database:** `data/memory/jarvis.sqlite` — created automatically on first API start.
- **localStorage cache:** `jarvis.session.v1` (active session id) and `jarvis.chat.v1` (message cache). Fallback only — SQLite is primary.
- **Gitignored:** `data/memory/` and `*.sqlite` rules both cover the database file.

## SQLite schema

```sql
CREATE TABLE chat_sessions (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  title      TEXT    NOT NULL DEFAULT 'New Chat',
  created_at TEXT    NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT    NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE chat_messages (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id INTEGER NOT NULL REFERENCES chat_sessions(id) ON DELETE CASCADE,
  role       TEXT    NOT NULL CHECK(role IN ('user', 'assistant', 'error', 'cancelled')),
  content    TEXT    NOT NULL,
  model      TEXT,
  created_at TEXT    NOT NULL DEFAULT (datetime('now'))
);
```

## API surface

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/chat` | Non-streaming chat (full response) |
| `POST` | `/chat/stream` | Streaming chat (NDJSON) |
| `GET` | `/ollama/status` | Ollama reachability + model list |
| `POST` | `/sessions` | Create session |
| `GET` | `/sessions` | List 50 most recent sessions |
| `GET` | `/sessions/:id` | Get session + all messages |
| `POST` | `/sessions/:id/messages` | Append a message |
| `PATCH` | `/sessions/:id` | Update session title |
| `DELETE` | `/sessions/:id` | Delete session + messages (CASCADE) |

## Known limitations

- No RAG or semantic memory.
- No file reading or editing tools.
- No voice input or output.
- No Home Assistant integration.
- No cross-device sync — SQLite is local only.
- No chat export.
- No multi-user support.
- Rename/delete buttons use CSS `group-hover` — invisible on touch/mobile.
- `Ctrl+Alt+N` does not check streaming state — safe because ChatPanel's unmount cleanup aborts in-flight requests.
- macOS `Cmd+Alt+N` not yet wired.
